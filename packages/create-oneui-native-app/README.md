# create-oneui-native-app

Scaffold a production-ready OneUI Native app in one command. Ships pre-wired with `@oneui/ui-native`, brand data CDN prefetch, JioType fonts, gesture handler, Reanimated, and Safe Area — ready to launch on iOS, Android, and (Expo only) web.

## Requirements

- **Node 18+**
- **Azure DevOps PAT** with read access to the `JIO-DS-OneUI-Native` package feed
- **iOS:** Xcode 15+ with Command Line Tools and at least one Simulator
- **Android:** Android Studio with an emulator or a USB-connected device

## Quick Start

```sh
npx create-oneui-native-app@latest
```

Or pass a project name directly:

```sh
npx create-oneui-native-app@latest my-app
```

## CLI Flags

| Flag | Description |
|---|---|
| `--template expo` | Expo Router template _(default)_ |
| `--template bare` | React Native CLI (bare) template |
| `--use-npm` | Force npm |
| `--use-yarn` | Force yarn |
| `--use-pnpm` | Force pnpm |
| `--use-bun` | Force bun |
| `--skip-install` | Skip dependency installation |
| `--skip-git` | Skip git init |
| `--skip-pat` | Skip PAT prompt; writes an `.npmrc` stub for manual editing |
| `--yes` / `-y` | Non-interactive CI mode (Expo template, skip PAT, all defaults) |

## Templates

### Expo (default)

Uses **Expo Router** for file-based routing and the Expo toolchain. Recommended for most teams — faster developer experience, built-in OTA updates, and web target support.

**Choose Expo when:**
- You want file-based routing out of the box
- You need web support (`expo start --web`)
- You prefer Expo's managed workflow (no manual Xcode/Gradle config)

### React Native CLI (bare)

Uses **React Navigation** with a standard `AppRegistry` entry point and the `@react-native/metro-config`. Gives you full control over the native layer.

**Choose bare when:**
- You need to integrate custom native modules that require native code changes
- Your team already manages `android/` and `ios/` directories directly
- You want zero Expo runtime dependency

> **Note:** The bare template does not include `android/` or `ios/` directories — these are generated by `react-native init`. See [Bare Template: Native Setup](#bare-template-native-setup) below.

## PAT Authentication

OneUI packages are hosted on a private Azure DevOps npm feed:

```
https://jio-dsp.pkgs.visualstudio.com/_packaging/JIO-DS-OneUI-Native/npm/registry/
```

The CLI will prompt you for a Personal Access Token (PAT) and write a `.npmrc` file in your project directory. Two input modes are supported:

1. **Raw PAT** — paste your token as-is; the CLI base64-encodes it automatically
2. **Pre-encoded PAT** — paste a token you have already encoded with `echo -n ":TOKEN" | base64`

The generated `.npmrc` sets `@oneui:registry` scoped to the feed, with `always-auth=true` and `strict-ssl=false` (required for this Azure DevOps endpoint).

If you use `--skip-pat` or `--yes`, the CLI writes an `.npmrc` stub with a `BASE64_ENCODED_PAT` placeholder that you replace manually.

## Brand Switching

1. Open `oneui.brands.json` and add or change a brand:

   ```json
   {
     "cdnUrl": "https://myjiostatic.cdn.jio.com/JDS/ReactNative",
     "brands": {
       "jio": "latest",
       "tira": "latest"
     }
   }
   ```

2. Fetch the brand data:

   ```sh
   npm run prefetch
   ```

3. Update `OneUIBrandProvider` in `_layout.tsx` (Expo) or `App.tsx` (bare):

   ```tsx
   <OneUIBrandProvider brand="tira" mode="light">
   ```

4. Restart the dev server.

## Development Commands

| Action | Expo | Bare |
|---|---|---|
| Start dev server | `npm run start` | `npm run start` |
| Launch iOS | `npm run ios` | `npm run ios` |
| Launch Android | `npm run android` | `npm run android` |
| Fetch brand data | `npm run prefetch` | `npm run prefetch` |
| Link font assets | _(handled by Expo)_ | `npm run link-assets` |
| Type check | `npm run typecheck` | `npm run typecheck` |

> All `start`, `ios`, and `android` scripts automatically run `npx oneui-native-cdn prefetch` before launching so brand data is always up to date.

## Next Steps After Scaffolding

```sh
cd my-app
npx oneui-native-cdn prefetch   # fetch brand data
npm run ios                      # launch on iOS simulator
```

Edit `oneui.brands.json` to add brands, then re-run `npm run prefetch` and restart the dev server to pick up changes.

## Bare Template: Native Setup

The bare template omits `android/` and `ios/` because these directories are large (>10 MB) and typically generated fresh. Set them up in two steps:

1. **Generate native directories** using the React Native community template:

   ```sh
   npx @react-native-community/cli init <AppName> --skip-install
   # copy android/ and ios/ from the generated project into your scaffold
   ```

2. **Register font assets** (required once after native dirs exist):

   ```sh
   npm run link-assets
   ```

   This copies JioType `.ttf` files into `android/app/src/main/assets/fonts/` and registers them in `ios/<App>/Info.plist` via `react-native-asset`.

3. **Install pods** (iOS only):

   ```sh
   cd ios && pod install && cd ..
   ```

4. **Run the app:**

   ```sh
   npm run ios      # or npm run android
   ```

## How It Works

The CLI is a self-contained Node.js binary (CommonJS, all dependencies bundled via tsup). When you run `npx create-oneui-native-app`:

1. Validates Node version (18+)
2. Prompts for project name (or reads from CLI arg)
3. Prompts for template choice (`expo` / `bare`) unless `--template` is set
4. Copies the chosen template from `templates/default/` or `templates/bare/`
5. Renames `_gitignore` → `.gitignore` (preserves `_layout.tsx` and similar Expo Router files)
6. Replaces `__PROJECT_NAME__` placeholders in `package.json` and `app.json`
7. Prompts for a PAT and writes `.npmrc` (unless `--skip-pat` / `--yes`)
8. Installs dependencies with the detected or specified package manager
9. Runs `git init` + initial commit
10. Checks for Xcode and `adb` and warns if either is missing
11. Prints next steps

## Contributing

Build the CLI:

```sh
pnpm build         # compiles src/index.ts → dist/index.cjs
pnpm typecheck     # zero errors expected
```

Smoke test without publishing:

```sh
node dist/index.cjs my-test-app --template bare --skip-install --skip-git --skip-pat
ls my-test-app/src/App.tsx        # should exist
rm -rf my-test-app

node dist/index.cjs my-expo-app --template expo --skip-install --skip-git --skip-pat
ls my-expo-app/src/app/_layout.tsx
rm -rf my-expo-app
```
