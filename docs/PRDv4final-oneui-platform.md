## 1. Executive summary

This PRD covers the full refactoring of OneUI Studio into a Turborepo monorepo with shared packages and a unified platform application. The work establishes the runtime architecture (ComponentHarness, ComponentMeta, AST system, code export) and the project structure that supports both the design system authoring surface (Studio) and the UI composition canvas (Builder) as route groups within a single Next.js app with seamless navigation.

### What we're building

A single platform at one URL where a user can navigate between design system authoring (configure brands, edit tokens, manage recipes) and experience building (compose UIs on a canvas, prompt AI to generate layouts, export code) without page reloads. Brand context persists across both surfaces. Components render with pixel-identical fidelity to Storybook everywhere in the platform.

### What we're not building (yet)

Phase 1-2 does not include: tldraw infinite canvas, AI generation, the campaign creator, or the property inspector panel. These are Phase 3+ features that plug into the foundation established here.

---

## 2. Constraints

- **No Tailwind.** CSS custom properties generated by the OKLCH engine. All styling via CSS variables.
- **Base UI.** Components are built on Base UI (headless) with CSS custom property styling.
- **Convex.** All brand data, component configs, recipes, and documents live in Convex with reactive subscriptions.
- **Next.js App Router.** The platform runs on Next.js App Router.
- **Turborepo.** Monorepo orchestration via Turborepo with pnpm workspaces.
- **Multi-brand.** Runtime brand switching via CSS custom property updates — no component tree re-render needed.

### 7.2 Renderer

```typescript
// packages/ast/renderer.tsx

import { memo } from 'react'
import { getComponentMeta } from '@oneui/meta'
import type { ASTNode, ComponentASTNode, TextNode, ExpressionNode } from './types'

export const ASTRenderer = memo(function ASTRenderer({ node }: { node: ASTNode }) {
  if (node.type === 'text') return <>{(node as TextNode).value}</>
  if (node.type === 'expression') {
    const expr = node as ExpressionNode
    return <span data-expression={expr.expression}>{expr.previewValue}</span>
  }

  const comp = node as ComponentASTNode
  const meta = getComponentMeta(comp.type)
  if (!meta) return <UnknownComponent type={comp.type} id={comp.id} />
  if (comp.meta?.hidden) return null

  const Component = meta.component
  const filteredProps: Record<string, any> = {}
  for (const [key, value] of Object.entries(comp.props)) {
    const propMeta = meta.props.find(p => p.name === key)
    if (propMeta?.type.kind === 'callback') continue
    filteredProps[key] = value
  }

  const children = comp.children?.map(child => <ASTRenderer key={child.id} node={child} />)
  const slotProps: Record<string, React.ReactNode> = {}
  if (comp.slots) {
    for (const [name, nodes] of Object.entries(comp.slots)) {
      slotProps[name] = nodes.map(child => <ASTRenderer key={child.id} node={child} />)
    }
  }

  return <Component {...filteredProps} {...slotProps} data-ast-id={comp.id}>{children}</Component>
})

function UnknownComponent({ type, id }: { type: string; id: string }) {
  return (
    <div data-ast-id={id} style={{
      padding: '8px 12px', border: '1px dashed var(--color-border-warning)',
      borderRadius: '4px', fontSize: '12px',
      color: 'var(--color-text-warning)', background: 'var(--color-background-warning)',
    }}>Unknown component: {type}</div>
  )
}
```

## 8. System D: Code export

```typescript
// packages/ast/compiler/ast-to-react.ts

import { getComponentMeta } from '@oneui/meta'
import type { ASTNode, ComponentASTNode } from '../types'

export function astToReact(nodes: ASTNode[], options = { includeImports: true, indent: 2 }): string {
  const imports = new Set<string>()
  const jsx = nodes.map(n => renderNode(n, 0, options.indent, imports)).join('\n')
  if (!options.includeImports) return jsx
  const importLines = Array.from(imports).sort().map(t => `import { ${t} } from '@oneui/ui'`).join('\n')
  return importLines + '\n\n' + jsx
}

function renderNode(node: ASTNode, depth: number, ind: number, imports: Set<string>): string {
  const pad = ' '.repeat(depth * ind)
  if (node.type === 'text') return `${pad}${(node as any).value}`
  if (node.type === 'expression') return `${pad}{${(node as any).expression}}`

  const comp = node as ComponentASTNode
  imports.add(comp.type)
  const meta = getComponentMeta(comp.type)

  const propEntries = Object.entries(comp.props).filter(([key, value]) => {
    if (meta) { const pm = meta.props.find(p => p.name === key); if (pm && pm.defaultValue === value) return false }
    return value !== undefined
  })

  const propsStr = propEntries.map(([k, v]) => {
    if (typeof v === 'boolean' && v) return k
    if (typeof v === 'string') return `${k}="${v}"`
    return `${k}={${JSON.stringify(v)}}`
  }).join(' ')

  const allChildren = comp.children ?? []
  if (allChildren.length === 0) {
    return `${pad}${propsStr ? `<${comp.type} ${propsStr} />` : `<${comp.type} />`}`
  }

  const open = propsStr ? `<${comp.type} ${propsStr}>` : `<${comp.type}>`
  const childStr = allChildren.map(c => renderNode(c, depth + 1, ind, imports)).join('\n')
  return `${pad}${open}\n${childStr}\n${pad}</${comp.type}>`
}
```

## 9. Sandpack decision matrix

| Use case | Renderer | Why |
|---|---|---|
| Component preview in editor | SharedRuntime | Must match Storybook |
| Component documentation | SharedRuntime | Same tokens, same visual |
| Campaign creator | SharedRuntime | Uses OneUI components with brand tokens |
| Landing page builder | SharedRuntime | Composes OneUI blocks |
| Freeform code sandbox | Sandpack | Arbitrary code outside component registry |
| External code preview | Sandpack | Non-OneUI dependencies need isolation |
| Embeddable playground (docs) | Sandpack | Public-facing, needs security isolation |

**Rule: OneUI components + brand tokens → SharedRuntime. Arbitrary user code → Sandpack.**

## 12. Technical risks and mitigations

| Risk | Impact | Mitigation |
|---|---|---|
| Package extraction breaks imports | Build fails | TypeScript path aliases during migration. Codemod for bulk rewriting. Run tsc after each extraction. |
| CSS `contain: layout` breaks token inheritance | Wrong rendering on canvas | CSS custom properties inherit through `contain: layout`. If issues, fall back to class-scoped reset. |
| Component side effects in shared runtime | Host conflicts | `CanvasContext.isCanvasPreview` flag. Components skip side effects in preview. |
| Performance with many canvas shapes | Sluggish rendering | BrandProvider global (pipeline once). Harness thin. ASTRenderer memoized. Phase 3 adds virtualization. |
| Metadata registry out of sync | Components missing from editor | Build-time warning for missing .meta.ts. CI check. Component still works — just not in visual editor. |
| Builder code leaks into Studio bundle | Bloated downloads | Next.js auto-splits per page. Verify with bundle analyzer. Heavy deps use dynamic(). |

## 14. Glossary

| Term | Definition |
|---|---|
| **Platform** | Single Next.js app containing Studio and Builder as route groups |
| **Studio** | `(studio)` route group — DS authoring |
| **Builder** | `(builder)` route group — UI composition canvas |
| **ComponentMeta** | Rich descriptor: type, props, slots, categories, brand overrides |
| **ComponentHarness** | Thin wrapper: Surface + density + CanvasContext. No BrandProvider. |
| **BrandProvider** | Global provider at app root. Runs OKLCH pipeline once. |
| **SharedRuntime** | Components in same React tree. No iframe. |
| **ComponentAST** | JSON tree using OneUI component types and props |
| **ASTRenderer** | Recursive component: AST nodes → real imports via registry |
| **Code export** | Compiler: AST → React source with imports |
| **Surface stacking** | `Surface` levels 0 → 1 → 2 shifting `--env-*` tokens |
| **OKLCH pipeline** | `buildAvailableScales → buildV4PaletteData → computeV4StackingForTheme → generateFilteredV4CSSFromSingleTheme` |
| **Turborepo** | Monorepo build orchestrator |
