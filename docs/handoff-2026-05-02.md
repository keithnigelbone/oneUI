**Effort:** 1–2 hr · **Owner:** anyone familiar with `useBrandCSS` · **Type:** test-quality fix

`packages/ui/src/engine/__tests__/fouc-prevention.test.ts` Test 3 + Test 7 currently `expect(source).toContain('if (!foundationData) return null')` against the source code of `useBrandCSS.ts`. The string-grep broke when `useBrandCSS` was refactored — function still behaves correctly, but the assertion's literal substring is gone.

**Fix path:** Render the hook with React Testing Library, drive it through the loading / mode=`none` / hydrated states, and assert on the **return values** instead of source text. Failing tests:

- `Test 3: useBrandCSS injection mode behavior > useBrandCSS returns '' for mode=none and null when data is loading`
- `Test 7: Preloader gate conditions > app-ready effect always gates on foundationData`
