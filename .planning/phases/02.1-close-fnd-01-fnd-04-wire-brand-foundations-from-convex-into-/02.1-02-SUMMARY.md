# Phase 02.1 Plan 02: Wire Convex-backed FoundationsLoader Into Run Route + Sub-brand Select Summary

**The run route now builds a Convex-backed `foundationsLoader` (Pitfall-1-safe overview→BrandFoundations map with optional sub-brand merge) and injects it into the Mastra workflow, while an optional dependent sub-brand Select threads `subBrandConfigId` end-to-end — closing FND-01 and the route-level half of FND-04.**

## Accomplishments
- Added `makeConvexFoundationsLoader(convex)` to `route.ts`: returns `null` for the `'jio'` placeholder and for a missing overview (→ engine system defaults, D-08); otherwise queries the PUBLIC `api.foundations.getBrandOverviewData` (A1), optionally merges a sub-brand via `applySubBrandAccentsToFoundation` (D-03/D-06) BEFORE the 3-field map, and maps `color?.config` VERBATIM (Pitfall 1).
- Extended `RunRequestBody.strict()` with `subBrandConfigId: z.string().optional()` (Pitfall 4 — `.strict()` rejects unknown fields), and injected `foundationsLoader` into `runExperienceWorkflow` only when `NEXT_PUBLIC_CONVEX_URL` is set, via conditional spread. `persistRun` untouched.
- Threaded `subBrandConfigId` through `PromptCardShape` (optional prop + `T.optional(T.string)` validator) and `useExperienceLabRun` (conditional-spread into the POST body, never an empty string).
- Added a dependent sub-brand `Select` to `RequestPanel`: `useQuery(api.subBrandConfigs.getByParentBrand, brandId ? { parentBrandId } : 'skip')` (disabled until a brand is chosen — D-02), `(Parent brand only)` as the first/default option, `handleSubBrandChange` clearing the prop on the empty value. Uses the project `Select` with token-only styling, reusing the brand-field markup.
- Re-exported `FoundationsLoader`/`FoundationsLoaderInput` from the agents barrel so the route can import the type by name.
- Proved both halves with extended tests: `runRoute.test.ts` (9 tests — loader injection, no-loader-when-unconfigured, `subBrandConfigId` forwarding, Pitfall-1 route-level map, sub-brand `getById` merge) and `requestPanel.test.tsx` (14 tests — render, skip-gating until brand chosen, population from `getByParentBrand`, default option, persist + clear).

## Decisions Made
- **Public `getBrandOverviewData` (A1, inherited):** the loader uses the public query, not `…Internal`; a backend HTTP client cannot invoke an `internalQuery`. The resolved data is identical (shared handler).
- **`color?.config` verbatim (Pitfall 1):** mapping `overview.color` (the foundation record) yields empty scales and a spurious gap; the resolver wants the `.config` sub-object.
- **Sub-brand merge before the map (D-03/D-06):** `applySubBrandAccentsToFoundation` runs only when `getById` returns a row, and returns base unchanged on a null arg — the optional sub-brand "just works".
- **No `getDefaultProps` entry for `subBrandConfigId`:** tldraw treats an absent optional as `undefined`; all shape/canvas tests pass without it (PATTERNS open item resolved as "leave out").
- **`persistRun` unchanged:** sub-brand persistence on the run record is out of FND-01/04 scope (research open question).
