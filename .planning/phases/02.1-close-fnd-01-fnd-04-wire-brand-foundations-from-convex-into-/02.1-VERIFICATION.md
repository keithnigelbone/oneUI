#### 2. CR-01 Robustness Decision — loader throw path

**Test:** Simulate or trigger a transient Convex error during foundation loading (e.g. network blip, malformed brandId that Convex rejects with a thrown error) and observe the run outcome.
**Expected (desired):** Run degrades gracefully to engine system defaults — `outcome === 'artifact'` via D-08 fallback, not `outcome === 'error'`.
**Expected (actual current behaviour):** The loader throws → propagates out of `resolveStep.execute` → caught by `runExperienceWorkflow`'s outer catch → `outcome === 'error'`, no artifact. The null-return path is D-08-compliant; the throw path is not.
**Why human:** CR-01 (flagged in 02.1-REVIEW.md) is an advisory robustness gap that does not block the wiring goal, but it means a transient Convex hiccup converts a generatable run into a hard failure rather than a graceful degradation. The developer must decide: fix before milestone close (wrap loader calls in try/catch returning null), or accept the current behaviour as-is given the design lab context where runs are user-initiated and retryable.
