```ts
function makeConvexFoundationsLoader(convex: ConvexHttpClient): FoundationsLoader {
  return async ({ brandId, subBrandConfigId }) => {
    if (brandId === PLACEHOLDER_BRAND_ID) return null;          // → system defaults (D-08)
    const overview = await convex.query(api.foundations.getBrandOverviewData, {  // see Pitfall 3 (internal vs public)
      brandId: brandId as Id<'brands'>,
    });
    if (!overview) return null;
    let base: Record<string, unknown> = overview;
    if (subBrandConfigId) {
      const sub = await convex.query(api.subBrandConfigs.getById, {
        id: subBrandConfigId as Id<'subBrandConfigs'>,
      });
      if (sub) base = applySubBrandAccentsToFoundation(overview, sub) ?? overview; // D-03
    }
    return {
      colorConfig:      (base as any).color?.config ?? null,    // ← VERBATIM brandCSSPrecompute.ts:43 (Pitfall 1)
      presetSelection:  (base as any).presetSelection ?? null,
      appearanceConfig: (base as any).appearanceConfig ?? null,
    };
  };
}
```

## Shared Patterns

### Overview → `BrandFoundations` Mapping (CRITICAL — Pitfall 1)
**Source:** `packages/convex/convex/brandCSSPrecompute.ts:42-49` (also `brandCSSExport.ts:42`)
**Apply to:** the route loader factory ONLY (never inside the agents package — D-05).
```ts
const input = {
  colorConfig:      foundationData.color?.config ?? null,   // ← .config, NOT the record
  presetSelection:  foundationData.presetSelection ?? null,
  appearanceConfig: foundationData.appearanceConfig ?? null,
  // typography/motion/grid/platforms/customFonts NOT needed by the resolver
};
```
`overview.color` is a foundation **record**; the resolver wants its `.config` sub-object. Mapping `colorConfig: overview.color` yields empty scales → `buildThemeConfig` returns `null` → spurious gap for a valid brand. Write `overview.color?.config ?? null` **verbatim**.

### Per-run function injection (DI idiom)
**Source:** `irGenerator.ts:62-73` (`__setCompileImpl`), `modelAdapter.ts` (`__setCallModelImpl`)
**Apply to:** the `foundationsLoader` type + `resolveStep`.
The existing seams are module-level (`__set*` swaps a process-wide impl). The `foundationsLoader` differs deliberately: it is **request-scoped** (carries a per-request Convex client), so it is injected through `RunExperienceInput` rather than a module global. Same testability — tests inject a plain mock function on the input object. Match the typed-function-with-clean-signature shape of the `__set*` idiom.

### Convex access at the route boundary (isolation guard — Pitfall 2)
**Source:** `route.ts:56-110` (`persistRun`)
**Apply to:** the loader factory.
`ConvexHttpClient` + `api` + `Id<'…'>` casts + `PLACEHOLDER_BRAND_ID` short-circuit + the `NEXT_PUBLIC_CONVEX_URL` guard all live in the route. The agents package MUST stay Convex-free — verified: `experience-builder-agents/package.json` has NO `@oneui/convex` dependency `[VERIFIED: package.json read]`. Never import `ConvexHttpClient` or the overview shape into the agents package.

### Conditional optional spread (no `undefined` in payloads)
**Source:** `workflow.ts:262`, `route.ts:72-75`, `useExperienceLabRun.ts` (pattern)
**Apply to:** every optional field added this phase (`subBrandConfigId`, `foundationsLoader`, the loader's `subBrandConfigId` arg).
```ts
...(value ? { key: value } : {})
```

### Sub-brand accent merge (no hand-rolling — D-03)
**Source:** `packages/shared/src/utils/subBrandFoundation.ts:50-79` (`applySubBrandAccentsToFoundation`)
**Apply to:** the loader, when `subBrandConfigId` is present, BEFORE the 3-field map.
Signature: `(baseData: Record<string,any> | null, currentSubBrand: SubBrandAccentFields | null) => Record<string,any> | null`. Returns `baseData` unchanged if either arg is null (optional sub-brand "just works"). The Convex `subBrandConfigs` row (`schema.ts:200-213`) is already `SubBrandAccentFields`-shaped (`subBrandFoundation.ts:8-16`) — pass `getById`'s result straight in, no remapping.

### Convex query in a panel (dependent variant)
**Source:** `RequestPanel.tsx:91` (`useQuery(api.brands.list)`)
**Apply to:** the sub-brand Select.
The sub-brand query is **dependent** — pass `'skip'` as the second arg until a brand is chosen (`useQuery(api.subBrandConfigs.getByParentBrand, brandId ? { parentBrandId } : 'skip')`). `undefined` while loading, `[]` when empty — never a fabricated list, exactly like the brand query's contract.
