## Pattern Assignments

### `_chat/ExperienceLabChat.tsx` (component host, streaming)

**Analog:** `packages/ui/src/components/ChatSurface/ChatSurface.tsx` (the primitive it consumes) — DO NOT rebuild chat; render `<ChatSurface>`.

**Import pattern** (Lab uses the `@oneui/ui-internal` deep-import alias → `packages/ui/src`; verified in `apps/platform/tsconfig.json:22` + `vitest.config.ts:14`):
```tsx
import { ChatSurface } from '@oneui/ui-internal/components/ChatSurface/ChatSurface';
// ChatSurface is ALSO exported from the @oneui/ui barrel (packages/ui/src/index.ts:759).
// Use the deep alias inside the Lab to match RequestPanel's deep-import convention.
```

**Core pattern — the headless props ChatSurface needs** (`ChatSurface.shared.ts:57-134`):
```tsx
<ChatSurface
  messages={messages}            // ChatMessage[] — { id, role, parts?: [{type, ...data}] }
  status={status}                // 'ready' | 'submitted' | 'streaming' | 'error'
  error={error}
  onSubmit={handleSubmit}        // (text: string) => void  — fires runForPrompt
  onStop={handleStop}            // abort the NDJSON controller
  onRegenerate={handleRegenerate}// re-run last prompt against focused artifact (D-02)
  greeting={greeting}
  suggestions={suggestions}      // SuggestionChip[] — empty-state chips (UI-SPEC copy)
  renderMessagePart={renderLabMessagePart}  // the D-09 dispatch
  composerProps={{ leadingInline: <ComposerControlStrip .../>, placeholder: 'Describe the experience to generate…' }}
  aboveComposer={focusedArtifactChip}        // D-02 focused-artifact indicator
  thinkingMessages={['Resolving foundations…', 'Retrieving components…', 'Planning…', /* … */]}
/>
```

**Transport rule (LOCKED anti-pattern):** ChatSurface is headless of transport. It only needs `messages[]`, `status`, `onSubmit`. Do NOT import `ai`/`@ai-sdk/react` or use `useChat` — that breaches the Lab-scoped single-`ai` gate. Wire the existing NDJSON `useExperienceLabRun` instead. `ChatMessage`/`UIMessage`/`ChatStatus` are declared structurally in `ChatSurface.shared.ts` precisely so `@oneui/ui` never pulls in `ai`.

**Brand options + artifact-type labels** (`RequestPanel.tsx:50-64, 122-131`):
```tsx

const brandOptions = (brands ?? []).map((b) => ({ value: b._id as string, label: b.name }));
const ARTIFACT_TYPE_OPTIONS = ARTIFACT_TYPES.map((t) => ({ value: t, label: ARTIFACT_TYPE_LABELS[t] }));
```

**CRITICAL — sub-brand skip-gate (lift verbatim, Pitfall 4 / RequestPanel.tsx:147-166):**
```tsx
const isResolvedBrandId = useMemo(
  () => Boolean(selectedBrandId) && (brands ?? []).some((b) => (b._id as string) === selectedBrandId),
  [brands, selectedBrandId],
);
const subBrands = useQuery(
  api.subBrandConfigs.getByParentBrand,
  isResolvedBrandId ? { parentBrandId: selectedBrandId as Id<'brands'> } : 'skip',  // 'skip' until resolved
);
```
> Naively lifting without the `'skip'` sentinel re-introduces the `ArgumentValidationError` crash 02.1 fixed — `getByParentBrand` validates `v.id('brands')`. The query MUST stay skipped until `brandId` resolves against the live list.

**Profile-options-by-type filtering** (`RequestPanel.tsx:170-177`):
```tsx
const profileOptions = useMemo(
  () => getValidProfilesForType(selectedType).map((p) => ({ value: p.id, label: p.label })),
  [selectedType],
);
```

**Type-change auto-snap of invalid profile** (`RequestPanel.tsx:202-218`) — keep this so the strip never holds an invalid type/profile pair.

**KEY DIFFERENCE from analog:** RequestPanel reads/writes shape props via `editor.updateShape`. The strip is FREE FORM STATE (React state / `useLabConversation`), NOT a tldraw shape. Replace every `updateProp(...) → editor.updateShape` with a `setState`. The selector JSX (`<Select value=… onChange=… options=… placeholder=… className=…>`) copies verbatim. Render inside `ChatComposer`'s `leadingInline` slot (`ChatComposer.shared.ts:47-51`).

- While `isRunning`: render `<AgentThinking messages={thinkingMessages} />` (rotating ticker; `AgentThinkingProps` accepts `messages?: ReadonlyArray<string>` + optional `icon`).
- On terminal frame: collapse to a one-line summary (`StreamingText text=… isStreaming={false}`), put the full `describeEvent` rows behind `Collapsible` ("Show run detail" / "Hide run detail", UI-SPEC copy).

**CRITICAL branch (Pitfall 5 / CHAT-06):** a `gap` result frame **carrying `result.campaignPlan`** is a *suspend*, NOT a failure. ### `_chat/parts/renderLabMessagePart.tsx` (utility dispatch, transform)

**Analog:** `packages/ui/src/components/ChatSurface/parts/index.tsx` `defaultRenderMessagePart` (lines 81-103) — the D-09 framework is exactly this `RenderMessagePart` slot.

**Pattern to copy** (`index.tsx:81-103` + `parts.shared.ts:24-30` const discriminators):
```tsx
import type { RenderMessagePart } from '@oneui/ui-internal/components/ChatSurface/ChatSurface.shared';

const PART_RUN_PROGRESS  = 'data-run-progress' as const;   // mirror the 'data-<name>' convention
const PART_CAMPAIGN_PLAN = 'data-campaign-plan' as const;

export const renderLabMessagePart: RenderMessagePart = (part, ctx) => {
  if (part.type === PART_RUN_PROGRESS)  return <RunTurn events={part.events} status={part.status} outcome={part.outcome} />;
  if (part.type === PART_CAMPAIGN_PLAN) return <CampaignPlanCard plan={part.plan} campaignRunId={part.campaignRunId} brandId={part.brandId} />;
  return null;  // null → ChatSurface falls through to built-in text/tool rendering
};
```
> `RenderMessagePart` signature: `(part: {type:string} & Record<string,unknown>, ctx: {message, index}) => ReactNode | null` (`ChatSurface.shared.ts:42-45`). Returning `null` is the documented fall-through. Part payloads follow the AI-SDK `data-*` convention (`parts.shared.ts:1-8`).

## Shared Patterns

### Deep-import convention (LAB-02/LAB-03)
**Source:** `RequestPanel.tsx:30-32`, `RunInspectorPanel.tsx:21-23`
**Apply to:** all new chat files
```tsx
import { Select } from '@oneui/ui-internal/components/Select/Select';
import { Button } from '@oneui/ui-internal/components/Button/Button';
import { Surface } from '@oneui/ui-internal/components/Surface/Surface';
import { Badge } from '@oneui/ui-internal/components/Badge/Badge';
```
Never import from the `@oneui/ui` barrel inside the Lab subtree (except where the existing canvas already does, e.g. `ExperienceLabCanvas.tsx:27` `@oneui/ui/components/Button`). Never import `(builder)`/`ExperienceCanvas` (LAB-03 eslint boundary). Never import `ai`/`@ai-sdk` (Lab single-`ai` gate) — only `modelAdapter.ts` may.

### Surface Context (MANDATORY — CLAUDE.md)
**Source:** `RunInspectorPanel.tsx:119` (`<Surface mode="subtle">`), design-composition skill §3
**Apply to:** run-turn container (`minimal`), inline plan card body (`subtle`), control-strip background (`minimal`), both panes (`default`)
- Never set `background` on a raw `<div>` holding Jio components — use `<Surface mode="…">`.
- No decorative strokes/borders on tinted Surfaces.
- Role-tinted override allowed via inline `style={{ '--Surface-Fill-Subtle': 'var(--Secondary-Subtle)' }}`.

### Typography V2 (MANDATORY — CLAUDE.md + UI-SPEC)
**Apply to:** all net-new wrappers (`RunTurn`, `CampaignPlanCard`, `ComposerControlStrip`, shell)
- Every text element: a role size token + its paired line-height token + a weight token + `font-family: var(--Typography-Font-Primary)` (or `-Code` for the event log).
- Body text `--Body-M-FontSize`/`-LineHeight`/`--Body-FontWeight-Low`; labels `--Label-S-*`/`--Label-FontWeight-Medium`; card titles `--Title-M-*`; badges `--Label-XS-*`; event log `--Code-S-*` + `--Typography-Font-Code`.
- Never legacy `--Typography-Size-*` / `--Typography-Weight-*`; never literal numbers.
- ChatSurface/ChatComposer/Select/Button/Badge already use V2 tokens — satisfied for free for reused components.

### Focus Halo (interactive — CLAUDE.md + UI-SPEC)
**Apply to:** net-new interactive elements (direction tiles, control-strip selects, send/confirm buttons)
```css
.el:focus-visible {
  outline: none;
  box-shadow:
    0 0 0 var(--Stroke-XL) var(--Surface-Halo-Gap, var(--Surface-Main)),
    0 0 0 var(--Focus-Outline-Width) var(--Focus-Outline);
}
```
Use `--Surface-Halo-Gap` (NOT `--Surface-Main` directly) so the inner gap adapts inside `<Surface>` containers. Where `@oneui/ui` owns focus styling, reuse it.

### Appearance type (CLAUDE.md)
**Apply to:** any new component exposing a multi-accent `appearance` prop
```ts
import type { ComponentAppearance } from '@oneui/shared';
```
Do not redefine the appearance union locally.

### Run-event semantic Badge mapping (reuse, don't re-author)
**Source:** `RunInspectorPanel.tsx:42-102` `describeEvent`
**Apply to:** `RunTurn.tsx` and any future run-progress surface.
