# Phase 04.1 Plan 05: Chat ‚Üî Canvas Selection Binding Summary

Two-way tldraw selection ‚Üî focused-turn binding (D-02 / CHAT-08): selecting an artifact card on the canvas focuses + scrolls to its originating chat turn and shows a neutral focus indicator; a turn's `Focus on canvas` action selects + zooms the linked shape; follow-ups iterate on the focused artifact with the run body unchanged.

## What Was Built

**Task 1 ‚Äî `useChatCanvasFocus.ts` binding hook (TDD):**
- Forward (canvas ‚Üí chat): the existing reactive `editor.sideEffects.registerAfterChangeHandler('instance_page_state' | 'shape', sync)` pattern (mirrored verbatim from `ExperienceLabCanvas.tsx:84-96`) reads `getSelectedShapeIds()`; a single selected `ArtifactCardShape` with a registered linkage sets `focusedArtifact` and calls `scrollToTurn(turnId)`. Non-artifact / multi / empty selections clear focus.
- Reverse (chat ‚Üí canvas): `focusOnCanvas(turnId)` maps the turn ‚Üí its linked shape and calls `editor.select(shapeId)` + `editor.zoomToSelection()`.
- Linkage: a bidirectional `Map` (`shapeToLink` / `turnToLink`) populated via `linkArtifact({turnId, shapeId, name, runNumber})` ‚Äî kept in refs so the reactive handler reads fresh associations without re-subscribing.
- Cleanup: the effect returns the two `registerAfterChangeHandler` unsubscribe fns; no `setInterval` / polling (T-04.1-14 perf posture identical to the shipped canvas sync).
- Filled `selectionBinding.test.tsx`: forward focus + scroll, reverse select+zoom, handler-cleanup/no-poll, and iterate-target-without-run-body-fields. The Focus-Halo CSS assertion goes green after Task 2.

**Task 2 ‚Äî Focused-artifact indicator + reverse action in the chat host:**
- `useChatCanvasFocus` wired into `ExperienceLabChat`; the shell now threads the tldraw `editor` down.
- `aboveComposer` slot renders a neutral `Chip` `Editing: {name} ¬∑ Run #{n}` + a `Clear focus` Chip, inside a `role="status" aria-live="polite"` region (visible AND announced).
- Each artifact assistant turn gets a `Focus on canvas` Chip (reverse binding) via a `renderMessagePart` wrapper that also stamps the `data-lab-turn-id` scroll anchor.
- D-12 linkage: an effect watches artifact-outcome assistant turns and links the Nth artifact turn to the Nth artifact shape on the canvas (the chat hook is the single 1:1 canvas caller).
- Iterate flow: when focused, the composer placeholder becomes `Iterate on {name}‚Ä¶` and `regenerate` targets the focused turn ‚Äî the posted `RunRequestBody` is byte-for-byte unchanged.
- Styles token-only: V2 label typography (`--Label-S-*` + `--Label-FontWeight-Medium`), `--Shape-Pill`, Focus Halo via `--Surface-Halo-Gap` (with the first-paint `--Surface-Main` inline fallback only). No `background` on raw containers ‚Äî the Chip carries its own neutral surface.
