### Human Verification Required

#### 1. Reload-safety — linked Convex artifacts restore on reopen

**Test:** Run a generation until an artifact card appears on the canvas. Then hard-reload the `/lab` route. Verify both (a) the chat conversation text rehydrates and (b) artifact cards appear on the canvas.

**Expected:** Chat messages from the previous session load from the Mastra InMemoryStore. Canvas artifact cards (backed by Convex `experienceArtifactVersions`) restore from the server.

**Why human:** The chat-message restore path is code-verified (InMemoryStore round-trip test). The canvas artifact restore path requires the full tldraw + Convex integration running against a live backend. The current implementation uses `InMemoryStore` which only survives within a single Node process — a cold serverless restart loses the chat thread. This is the documented single-instance durability boundary. The human test should confirm behavior within a single Node process and document the durability boundary.

#### 2. Streaming run turn — visual ticker and collapse

**Test:** Trigger a generation run from the chat composer. Observe the RunTurn part in the chat stream.

**Expected:** While running, an in-place rotating ticker shows step names (Resolving foundations, Retrieving components, …). On completion it collapses to a one-line summary text. A "Show run detail" toggle reveals the full ordered event list.

**Why human:** Visual animated behavior over a live NDJSON stream; structural rendering is unit-tested but the live progressive update UX needs a browser.

#### 3. Campaign-plan HITL — inline card resumes workflow and frames appear on canvas

**Test:** Trigger a `instagram-carousel` or `social-post` run (campaign branch). Observe the inline `CampaignPlanCard` appearing in the chat stream. Select a creative direction and frame count. Click "Generate frames".

**Expected:** The card renders the brief, audience, message hierarchy, and 3 direction tiles with the recommended one marked. After clicking "Generate frames", the `/resume` route is called and carousel frame cards appear on the canvas.

**Why human:** Requires a live Mastra workflow with real model to reach the campaign-plan suspend checkpoint.

#### 4. Chat ↔ canvas two-way selection binding

**Test:** After a generation completes and an artifact card is on the canvas, click the artifact card. Then use a "Focus on canvas" action from the chat turn.

**Expected:** Clicking the canvas card scrolls the chat to its originating turn and shows "Editing: {name} · Run #{n}" in the composer area. The "Focus on canvas" turn action selects and zooms to the artifact card.

**Why human:** tldraw selection + DOM scroll + indicator update requires a real browser with canvas mounted.

#### 5. stop() cleanly cancels a live mid-stream read

**Test:** Submit a generation and click the stop button while the run is actively streaming (ticker is animating).

**Expected:** The stream stops within ~1 second. Status returns to 'ready'. The run turn shows a neutral state (no error headline). A subsequent submit works normally.

**Why human:** The CR-01 fix threads the abort signal into `readNdjson` to cancel the `ReadableStream` reader. This requires a real live stream to test the abort-during-streaming path (the unit tests cover the mocked fixture path with `signal?.aborted` checks, but the actual `reader.cancel()` call needs a live connection).
