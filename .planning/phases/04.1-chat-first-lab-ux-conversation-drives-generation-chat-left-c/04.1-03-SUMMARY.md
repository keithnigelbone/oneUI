# Phase 04.1 Plan 03: Chat-first Lab structural restructure Summary

**Left chat + right canvas Lab shell where the composer drives generation over the unchanged NDJSON run contract, a streaming run-turn part tickers→summarises→expands, and the chat hook is the single caller of canvas artifact/gap placement.**

## Performance

- **Duration:** ~18 min
- **Tasks:** 3 (2 TDD)
- **Files modified:** 15 (8 created, 7 modified)

## Accomplishments
- Stood up the chat-first layout (D-13): `ExperienceLabShell` is a token-only CSS-grid split — left `ExperienceLabChat` (`minmax(var(--Spacing-40), 30%)`) + right `ExperienceLabCanvas` (`1fr`); both docked panels are retired.
- Built `useLabConversation` — the chat transport/state hook. It reuses `readNdjson` + the `RunStreamFrame` contract verbatim, POSTs the UNCHANGED `RunRequestBody` from composer-strip state (D-01/CHAT-01), accumulates events into a `data-run-progress` part, persists each turn through the Plan-01 conversation route, rehydrates on mount (D-06), and is the single caller of the canvas placement callbacks on terminal frames (D-12). No `ai` import (CHAT-11).
- Built the streaming `RunTurn` part (D-11): a live step ticker (`AgentThinking`, `aria-live="polite"`) → collapsed `StreamingText` summary → expandable `describeEvent` detail (reused verbatim from `RunInspectorPanel`, Code typography role). Branches on `result.campaignPlan` BEFORE treating a gap as failure so a campaign suspend never renders a refusal (D-12/CHAT-06); only a real gap renders the `aria-live="assertive"` refusal headline.
- Wired `renderLabMessagePart` (D-09 dispatch): routes `data-run-progress` → `<RunTurn>`, falls through to `null` for unknown parts, and reserves a documented `data-campaign-plan` extension point for Plan 04.
- Adapted `useExperienceLabRun` + `ExperienceLabCanvas`: kept the artifact-card / Run #N frame / gap-card placement (CHAT-02/D-12), exposed them as `canvasCallbacks`, removed the prompt-card entry branch (anchors to the viewport instead), removed the `dockRight` panels, and preserved the selection-observation handler for Plan 05.

## Deviations from Plan

None — plan executed as written. Two small implementation choices worth noting (both within plan intent):

- **Test design for the single-ai gate:** the plan's acceptance criterion is a raw grep for `from 'ai'`/`@ai-sdk/react`/`useChat`. A naive substring grep matched the anti-pattern documentation in the file headers (which legitimately name those tokens). The test strips comments before asserting on import statements / `useChat(` calls — preserving the intent (no real `ai` import) without a false positive. The authoritative gate `pnpm check:single-ai` passes.
- **Responsive single-column stacking** (a UI-SPEC *recommendation*, not a must-have) was deferred from the shell module: a literal breakpoint width would trip `check:literals` over the Lab route, and the plan's must-haves only require the left/right split. Documented in the CSS module header.
