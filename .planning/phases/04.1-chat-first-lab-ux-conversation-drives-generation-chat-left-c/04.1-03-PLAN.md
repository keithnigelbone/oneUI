The campaign-plan inline card and the chat↔canvas selection binding are added in Wave 3 (Plans 04, 05) on top of the dispatch + host this plan creates.

Scope note (cohesion — deliberately NOT split): the shell, the conversation hook, and the canvas adaptation are one structural unit. Task 3 threads the `canvasCallbacks` ({ placeArtifact, flipToGapState }) from the canvas through `ExperienceLabShell` → `ExperienceLabChat` → `useLabConversation` (built in Task 1), so the shell/canvas work is tightly coupled to this plan's chat host and would not stand alone in a separate Wave-2 plan without re-wiring the same prop seam. Keeping it intact avoids a cross-plan callback contract for a single wave; the 10 files are token-only CSS modules + their TSX pairs plus the existing canvas/run files, all within the ~50% context budget.
</objective>

| T-04.1-07 | Tampering (Stored XSS) | run-turn rendering of streamed text | mitigate | RunTurn renders text via React escaping (StreamingText), never `dangerouslySetInnerHTML`; event-log rows are structured `describeEvent` labels, not raw markup. |
