### Documented Contract Correction (not a code deviation)

The plan assumed `@mastra/core/storage` `FilesystemStore` could provide single-instance durability for the `memory` domain. In `@mastra/core@1.37.1`, `FilesystemStore` implements **only the 7 editor domains** (agents/promptBlocks/scorerDefinitions/mcpClients/mcpServers/workspaces/skills) — **not** the `memory` domain — so it cannot back conversation messages. The store therefore uses `InMemoryStore`, which gives **single-process durability** (threads survive across requests within one Node process). Cross-process / multi-instance durability is the deferred upgrade: the documented Convex `experienceChat` table, or a DB-backed Mastra store such as `@mastra/libsql` (gated behind the package-legitimacy checkpoint if ever adopted).
