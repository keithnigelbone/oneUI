## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → conversation route | client-generated sessionId + message content cross into the Node store |
| conversation store → filesystem | thread/message data persisted to an app-owned directory |
| package install boundary | new npm dependency entering the dependency tree |

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|

| T-04.1-02 | Information Disclosure | persisted conversation messages | mitigate | Never persist raw request headers/env/ANTHROPIC_API_KEY into the thread; only role+content+structured part data. |
| T-04.1-03 | Spoofing/Elevation | thread ownership (sessionId/resourceId) | mitigate | `query` is scoped by threadId; Test 3 asserts thread A cannot read thread B. sessionId is opaque, non-auth (ASVS V3 partial). |
| T-04.1-SC | Tampering | npm install (`@mastra/memory`/`@mastra/libsql`) | mitigate | `[ASSUMED]` packages gated behind the blocking-human checkpoint; verify on npmjs.com; install in apps/platform only. |
