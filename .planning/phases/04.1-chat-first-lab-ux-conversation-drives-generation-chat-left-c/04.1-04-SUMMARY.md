# Phase 04.1 Plan 04: Inline Campaign-Plan Card Summary

Built the inline interactive campaign-plan HITL card (D-08) ‚Äî the named deliverable of Phase 04.1 ‚Äî rendered in the chat stream via the D-09 inline-card framework's second consumer, posting a direction + frame-count selection to the unchanged `/api/experience-lab/resume` route (D-10) and feeding the resume NDJSON stream back to the canvas placement seam (D-12).

## What Was Built

- **`CampaignPlanCard.tsx`** ‚Äî a `<Surface mode="subtle">` grouped card composed from deep `@oneui/ui-internal` imports (`Surface`, `RadioGroup`/`Radio`, `Stepper`, `Button`, `Badge`). It renders the full `CampaignPlanT`: Brief, Audience, Message hierarchy, and the 3 creative-direction tiles (each showing concept + `copyAngle ¬∑ leadRole ¬∑ surfaceMood`) with the recommended tile marked by a single `Badge appearance="primary"`. A `Stepper` (1..10) labelled "Frames per direction" and one `Button variant="bold" appearance="primary"` "Generate frames" complete the HITL. On confirm it POSTs the exact `ResumeRequestBody` and hands the response to `onResolved`.
- **`renderLabMessagePart.tsx`** ‚Äî extended at the Plan-03 extension point with the `data-campaign-plan` branch ‚Üí `CampaignPlanCard`, plus a `CampaignPlanPart` payload type. The `null` fall-through for unknown parts and the `data-run-progress` ‚Üí `RunTurn` branch are untouched (CHAT-07 proof: the D-09 framework absorbs a second consumer without refactor).
- **`CampaignPlanCard.module.css`** ‚Äî token-only V2: Surface-driven tint (no raw `background`, no decorative stroke), Title/Body/Label roles each paired with line-height + weight + `font-family: var(--Typography-Font-Primary)`, f-step spacing/shape, Focus Halo via `--Surface-Halo-Gap` on the focusable heading.

## a11y

- The card is a labelled `role="group"` (`aria-labelledby` the "Campaign plan" title).
- Focus moves to the heading on mount so keyboard users land on the checkpoint.
- The direction `RadioGroup` and frame-count `Stepper` are fully keyboard-operable (component-owned).

## Verification

- `pnpm --filter @oneui/platform test -- campaignPlanCard runTurn` ‚Äî green (10/10). Full `experience-lab` suite: 12 files pass, no regressions.
- Resume body asserted to be EXACTLY `{runId,brandId,directionIndex,frameCount,subBrandConfigId?}` with `subBrandConfigId` conditionally omitted.
- Plan fields render React-escaped (no `<script>` node, no `dangerouslySetInnerHTML`) ‚Äî T-04.1-12.

- No `(builder)`/`ExperienceCanvas` import, no `ai`/`@ai-sdk` import in the card.
