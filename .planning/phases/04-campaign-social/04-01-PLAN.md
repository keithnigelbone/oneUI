<objective>
Light up the non-web half of the foundation resolver — the spine of Phase 4. Today `isCoveredProfile` hard-gates every `coverage: 'assumed'` (non-web) profile straight to a gap. This plan adds a deterministic, foundation-backed non-web branch: a typed Lab map (`outputProfile → { platformId, breakpointId }`), a resolver branch that looks the target up in the brand's `PlatformsFoundationConfig`, computes dimensions/type-scale/safe-area via the EXISTING `@oneui/shared` dimension helpers on a hit, and returns the typed FND-03 gap (no dimension numbers) on a miss.

This is MVP vertical-slice task 1: after it, the campaign branch can ask "what are this canvas's real dimensions?" and get either real foundation-resolved dims or an honest gap — the contract every later slice (planner, carousel, export) builds on. No invented dimensions, ever (D-01/D-02/D-03).

Purpose: FND-02 + CAMP-05 — resolver covers non-web profiles; frames resolve dims/type-scale/safe-areas from Jio foundations before generation.
Output: Extended `foundationResolve` contract, new `profilePlatformMap`, extended `foundationResolver`, exported `mmToPx`, unit tests for hit + miss.
</objective>
