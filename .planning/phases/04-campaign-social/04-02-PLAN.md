<objective>
Make the campaign brief → planner → HITL selection a first-class path. When the artifact type is `social-post`/`instagram-carousel`, the prompt card reveals brief fields (audience/objective/channel — D-04, no new card kind), the run takes the campaign branch, a new `runCampaignPlanner` produces an Anthropic-safe DS-grounded plan (brief summary + audience + message hierarchy + exactly 3 creative directions + recommended index + frame count — CAMP-02/D-06), and the workflow SUSPENDS at a campaign-plan checkpoint carrying the plan on `suspendPayload`. Before the workflow suspends, the `CampaignPlan` payload is PERSISTED to Convex (append-only, keyed by `runId`) so a new resume route reads it back from Convex — not from process-memory cache, which does not survive between the run POST and the resume POST in serverless/multi-process (Pitfall 4 / A5). The resume route applies the user's `{ directionIndex, frameCount }` selection (D-05) — branching lives in the workflow, never a model callback (ORCH-04). Frame generation itself is plan 03.

This is MVP vertical-slice task 2: after it, a user can enter a brief, run, see the planner's directions surfaced (via `suspendPayload`), and POST a selection that re-enters the workflow — a complete plan-and-pick slice on top of plan 01's foundation resolution. The non-web branch only enters the planner AFTER the resolver confirms a covered canvas (on a foundation miss it short-circuits to the FND-03 gap from plan 01).

Purpose: CAMP-01 (first-class campaign path), CAMP-02 (planner output). The HITL select + resume seam this plan builds is what CAMP-03 depends on, but CAMP-03 itself (per-frame ToV headline/body/CTA + caption copy) is DELIVERED in plan 03 where the `voiceAdapter` call lives — so CAMP-03 is owned by 04-03's `requirements`, not this plan.
Output: `campaignPlan` schema, `runCampaignPlanner`, brief-field UI + run-body extension, campaign-plan suspend/resume checkpoint, Convex plan persistence before suspend, + resume route that reads the plan from Convex.
</objective>
