<objective>
Generate the carousel itself. After the user picks a direction (plan 02's resume), drive N ordered frames through the EXISTING per-frame quality loop: each frame resolves its dimensions/type-scale/safe-areas from the foundation (plan 01), gets per-frame headline/body/CTA + caption copy from the existing ToV adapter (CAMP-03), then runs generate→compile→validate→evaluate→bounded-repair so it is provably DS-compliant on its own (D-09). Frames share one `variantGroupId` and carry a sequential `orderIndex` (D-07 — order is meaningful, unlike interchangeable variants). A carousel-level SHARED budget accumulator (Pitfall 3) caps repair attempts/tokens across all frames, not per-frame. Frames render as an ordered tldraw group, left-to-right, "Frame n of N".

This is MVP vertical-slice task 3: after it, a full campaign run produces a real, grouped, ordered, DS-compliant carousel on the canvas — the visible payoff of the brief→plan→pick→generate path.

Purpose: CAMP-03 (per-frame headline/body/CTA + caption copy via the existing ToV adapter — DELIVERED here, in the carousel driver's per-frame loop), CAMP-04 (ordered IR-backed carousel frames grouped on canvas), CAMP-05 (each frame resolves foundation dims before generation). 04-02 builds the HITL/resume seam CAMP-03 depends on but does not itself wire ToV.
Output: additive `orderIndex` schema field, `carousel.ts` driver + shared-budget wrapper, workflow campaign-branch wiring after resume, ordered tldraw group frame.
</objective>
