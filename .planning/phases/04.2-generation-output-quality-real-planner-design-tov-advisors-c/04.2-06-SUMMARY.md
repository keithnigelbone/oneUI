# Phase 04.2 Plan 06: Canvas Per-Agent Activity Cursors (AGENT-04 / D-06c) Summary

Built the highest-risk, net-new leg of the transparency layer: the canvas per-agent activity cursors. The Task-1 decision checkpoint locked the substrate as **`postmessage-rects`** (user selection), and Task 2 implemented exactly that path. Each compiled IR node now carries a stable `data-ir-node-id`; the sandboxed first-party preview iframe reports each node's bounding rect to the canvas via a rect-only `postMessage`; and the artifact card draws lightweight per-agent pointers OUTSIDE the iframe from the Plan-05 `agentTrace` (Design → layout regions, Tone → copy/CTA sections, Evaluator → backfilled/blocked nodes). The implementation is strictly additive — the live preview render, the LOCKED CSP / separate-origin isolation (PREV-01), and the canvas perf budget are all untouched.

## Substrate Decision (Task 1 — resolved at checkpoint)

**Locked: `postmessage-rects`.** Rationale (from the plan's options): reuses the existing iframe preview; node ids already exist in the IR; the iframe reports per-node bounding rects via `postMessage` to the canvas, which draws the overlay outside the iframe with NO CSP relaxation. The alternatives (`canvas-side-overlay-only` — coarser section-only fidelity; `defer`) were not selected.

## What Was Built (Task 2 — `3e05b7a9`)

### The substrate
- **`irToAst.ts`** — new `withIrNodeId(props, id)` helper stamps `data-ir-node-id` into the prop bag of every compiled node: registry instances (`instance.id`), layout primitive nodes (computed `node.id`), legacy section `Stack` wraps + section-anchored layout roots (`section.id`), and the artifact root (`exp-root`). The recursive builder still emits ONLY `kind:'component'`/`kind:'text'` — no `element` node, no markup. The attribute is a generated id string (T-04.2-15). All 52 existing irToAst tests stay green (the markup-free / no-element assertions are unaffected).
- **`ASTRenderer.tsx`** — added `data-ir-` to `ALWAYS_ALLOWED_PREFIXES` so the stamped id survives the leaf prop sanitizer (which otherwise drops props not matching `data-ast-`/`aria-`/`data-oneui-`) and lands on the rendered DOM element of leaf components too. astToReact (22) + canvas (9) suites stay green.

### The rect-only channel
- **`RenderASTBranded.tsx`** — new `IrNodeRectReporter` runs INSIDE the first-party AST-render iframe content: it reads every `[data-ir-node-id]` element's `getBoundingClientRect`, rounds the coords, and posts `{ type: 'oneui:ir-node-rects', rects: [{id,left,top,width,height}] }` to `window.parent`. rAF-batched; re-reports on window resize, capture-phase scroll, and a `ResizeObserver` on the render target; an initial + 300ms-settle report covers async font/brand-CSS layout. It only posts when actually embedded (`window.parent !== window`). **The payload is node-id strings + numbers ONLY — it never reads `document.cookie`/`localStorage`/any auth surface** (T-04.2-14).

### The agent->node->region model + overlay
- **`agentCursors.ts`** (new, pure) — `IR_NODE_RECTS_MESSAGE_TYPE` + `IrNodeRect`/`IrNodeRectsMessage` contract; `parseIrNodeRectsMessage` (rejects any non-rect-shaped cross-frame message — defence in depth); `agentCursorsFromTrace` (pure: `agentTrace` → per-agent `{agent,label,appearance,nodeIds}` — Design from `designRecs[].sectionId`, Tone from `toneRecs[].sectionId`, Evaluator from `backfilled[].instanceId` else blocked-section fallback else nothing); `resolveCursorRegions` (joins cursors against reported rects, dropping ids with no rect).
- **`ArtifactCardShape.tsx`** — new `AgentCursorOverlay` rendered inside the `live` `PreviewRegion`, absolutely positioned OVER (never inside) the iframe with `pointerEvents:'none'`. It listens for the rect message (only while a trace produced cursors), resolves regions, and draws an outlined box + a tiny role-tinted label per touched region. Token-only styling: borders/shadows/labels use `--{Role}-TintedA11y`/`--{Role}-Subtle` + `--Stroke-M/S` + `--Label-XS-*` + `--Shape-2` — no literals. Renders NOTHING when there is no trace or no rects yet (backward-compatible + graceful while the preview escalates to live).

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] The stamped `data-ir-node-id` would be dropped before reaching the DOM**
- **Found during:** Task 2 (tracing the data-attribute from irToAst through the render path).
- **Issue:** ASTRenderer's leaf prop sanitizer drops any prop not matching `ALWAYS_ALLOWED_PREFIXES` (`data-ast-`/`aria-`/`data-oneui-`). A `data-ir-node-id` stamped in irToAst would therefore be silently stripped off leaf components (Badge/Button/…), so the iframe reporter would find no rect for those nodes — the substrate would be inert.
- **Fix:** Added `data-ir-` to `ALWAYS_ALLOWED_PREFIXES` in `ASTRenderer.tsx` so the id survives onto every rendered DOM element. This is the one genuinely-required seam file beyond the plan's named `irToAst.ts` / `ArtifactCardShape.tsx` / `agentCursors.ts`.
- **Files modified:** `packages/ui/src/runtime/ASTRenderer.tsx`.
- **Commit:** `3e05b7a9`.

**2. [Rule 3 - Blocking] The rect reporter needs a host inside the first-party preview iframe**
- **Found during:** Task 2 (the plan's `postmessage-rects` path requires a rect-reporting script INSIDE the rendered output, but the plan's file list named only the IR mapper + the two canvas files).
- **Issue:** The iframe content is `RenderASTBranded` at `/internal/render-ast`; without a reporter mounted there, no rects are ever posted and the overlay has nothing to draw.
- **Fix:** Added `IrNodeRectReporter` to `RenderASTBranded.tsx` (the trusted first-party AST-render path ONLY — never the untrusted compiled-bundle/Daytona path, whose sandbox stays strict `allow-scripts`). Reports rects only when embedded; carries ids+numbers only.
- **Files modified:** `apps/platform/src/app/internal/render-ast/RenderASTBranded.tsx`.
- **Commit:** `3e05b7a9`.
