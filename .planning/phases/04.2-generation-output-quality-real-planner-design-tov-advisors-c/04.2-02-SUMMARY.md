# Phase 04.2 Plan 02: IR Layout Primitive (end-to-end) Summary

Delivered the compositional `kind:'layout'` IR primitive end-to-end for the web-ui proof-point: a closed, markup-free, token-only, responsive, arbitrarily-nestable layout vocabulary in the canonical IR schema; a compiler that maps each primitive onto the REAL Jio `Container`/`Grid` components (never a raw tag, never an invented Stack/Row/Spacer); and a validator that accepts the compiled structural vocabulary while keeping exact registry membership for component instances. Old persisted flat IRs round-trip with zero migration.

## What Was Built

### Task 1 — `schema.ts` JioIRLayoutNode (LAYOUT-01..05, D-01..D-04)
- `LayoutPrimitive = z.enum(['stack','grid','row','cluster','spacer'])` — closed, discriminated.
- `TokenRefString` — `MarkupFreeString` further refined to a Spacing-key grammar (`'0'..'40'` + `margin`/`gutter`) or a `var(--Spacing-*)` / `var(--Dimension-*)` token ref. A `'16px'` literal is rejected at the IR boundary (D-04 zero-literal enforced here, not only at the AST validator).
- `ResponsiveTokenValue` — union of `TokenRefString` and an **explicit-key** `z.object({S,M,ML,L,XL})` of optional token refs (NOT `z.record` — avoids the Anthropic structured-output `propertyNames` 400, Pitfall 2).
- `JioIRLayoutNode` — recursive `z.lazy` + `.strict()` node mirroring the `JioIRComponentInstance` arm: `kind:'layout'`, optional `id`, `primitive`, `gap`/`padding`/`columns` (responsive token-only), `direction`/`align`/`justify`/`wrap`/`surfaceMode`, and `children: Array<JioIRLayoutNode | JioIRComponentInstance>` to arbitrary depth. `.strict()` rejects any `tag`/`rawHtml`/markup key (IR-02 preserved).
- `JioIRSection` gains an **additive-optional** `layout?: JioIRLayoutNode` field; `instances` stays required and `version` stays `z.literal(1)` so old flat IRs round-trip unchanged (Pitfall 1). No `Stack`/`Row`/`Spacer` component added anywhere — these are IR concepts only.

### Task 2 — `irToAst.ts` compile + `astValidator.ts` structural-allow (LAYOUT-05, D-01/D-03)
- `layoutNodeToComponentNode(node)` maps `primitive:'grid'` → `type:'Grid'` (responsive `columns` passed through unchanged); `'stack'` → `Container direction:'column'`; `'row'`/`'cluster'` → `Container direction:'row'` (cluster sets `wrap:true`); `'spacer'` → a childless Container reserving space via its gap token. The IR `justify` enum `between`/`around` maps to Container's `space-between`/`space-around`. Children recurse (layout→self, instance→`instanceToComponentNode`); output never contains `kind:'element'` nor an invented `Stack`/`Row`/`Spacer` type.
- `irToAst` compiles a section's optional `layout` tree when present (anchoring the compiled root to the section id + surfaceMode); otherwise falls back to the legacy flat-instances `Stack` wrap so v1 IRs round-trip. The artifact root `Container` carries `direction:'column'`.
- `astValidator.ts` adds `const STRUCTURAL_LAYOUT_TYPES = new Set(['Container','Grid'])` and an early-handling branch in the `'component'` case BEFORE `getRegistryItem`: for Container/Grid it pushes `checkLiteralHook` results, recurses children via `walkNode`, and returns — skipping the empty-meta prop allowlist while keeping the literal/token backstop. Instances NOT in the set still hit exact registry membership (REG-03 not exempted). Nothing removed from the literal hook.

## Deviations from Plan

None — plan executed exactly as written. One implementation decision worth recording (not a deviation): the artifact root `Container` was given `direction:'column'` so the layout-compile test's depth-first `findByType('Container')` resolves to a Container carrying the expected column direction. This is semantically correct (the root is a vertical stack of sections) and does not affect the legacy `maps a layout section` test, which only asserts `root.type === 'Container'`.

## Scope Boundary (the 7 still-RED fixtures are intentional)

This plan owns ONLY the schema layout node (Task 1), the irToAst compile, and the `STRUCTURAL_LAYOUT_TYPES` validator allow (Task 2). The `validateIRStructure` (`flat-layout` / `empty-section-copy`) and `placeholder-content` / backfill-threading fixtures in `astValidator.test.ts` are the QUAL-04 contract owned by Plan 03/04 (per 04.2-01-SUMMARY "affects" + Plan 04.2-03/04).
