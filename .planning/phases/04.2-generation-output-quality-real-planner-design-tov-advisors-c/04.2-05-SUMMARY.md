# Phase 04.2 Plan 05: Multi-Agent Transparency Layer (parts a/b/d) Summary

Made the Planner/Design/ToV/Assembler/Compiler/Validator/Evaluator pipeline legible — each agent now leaves a visible, persisted trace. The agent-step trace persists additively on append-only Convex per artifact version (AGENT-01/D-06a), the 04.1 chat run-turn ticker shows named human-readable agent activity lines (AGENT-02/D-06b), and the canvas artifact card exposes an expandable "How this was built" disclosure surfacing planner output, design recs, ToV recs, registry matches, validation result, and eval score (AGENT-03/D-06d). The principle held throughout: transparency, not noise — legible without implementation knowledge, the final IR stays the single source of truth. Part c (canvas agent cursors) remains Plan 06.

## What Was Built

### Task 1 (`c8540f44`) — Persist the trace additively + assemble it in finalizeRun (AGENT-01)
- `schema.ts`: additive `agentTrace: v.optional(v.any())` on `experienceArtifactVersions`, mirroring the `evaluation`/`previewState` precedent (append-only, NO migration; structured agent outputs only, NO auth/session/Convex token — T-04.2-11).
- `experienceRuns.ts`: `persistArtifact.args` gains `agentTrace: v.optional(v.any())`, written into the `experienceArtifactVersions` insert.
- `runTypes.ts`: typed `AgentTraceT` (`planner` {sections, messageHierarchy, primaryCTA} / `designRecs` / `toneRecs` / `registryMatches` / `validation` {passed, blockingCodes} / `evaluation` {composite} / `backfilled`) added optionally to `RunExperienceResult`.
- `workflow.ts`: `assembleAgentTrace(ctx)` builds the trace from `ctx.plan` / `ctx.designSpec` / `ctx.copySpec` / `ctx.ir` (distinct component types = registry matches) / `ctx.validation` (passed + blocking codes) / `ctx.evaluation` (composite) / `ctx.backfilled` — every field optional so a partial run still produces a legible trace; returns `undefined` when nothing traceable ran (old/empty runs round-trip). Wired into `finalizeRun`'s result spread.
- `index.ts` + `workflow.ts`: `AgentTraceT` re-exported from the agents barrel.

### Task 2 — Push the additive Convex schema (live deployment)
- `npx convex dev --once` from `packages/convex` reported "Convex functions ready!" with NO migration prompt (additive `v.optional` requires none). The live dev deployment now carries the `agentTrace` field, so downstream typecheck/verification reads the real schema (not a stale false-positive). This was the project's dev push — NOT a production deploy (the orchestrator handles the post-phase schema-drift check).

### Task 3 (`bbced848`) — Chat step labels (AGENT-02) + "How this was built" trace card (AGENT-03)
- `RunTurn.tsx`: `STEP_LABELS` map (`plan`→"Planner is defining structure", `design`→"Design advisor is selecting layout, hierarchy & components", `copy`→"Tone advisor is refining copy & CTA", `generate-ir`→"Assembler is producing valid One UI IR", `compile`→"Compiler is validating React + Jio CSS", `validate`→"Validator is checking Jio compliance", `evaluate`→"Evaluator is checking screenshot quality", `repair`→"Repairing the IR"); `describeEvent`'s `'step'` case maps `event.step` through it with a raw-token fallback. Reuses the existing 04.1 ticker — no run-turn refactor.
- `runStream.ts`: `RunResultFrame` gains optional `agentTrace` (typed `AgentTraceT`), carried exactly like `evaluation`/`previewState`.
- `ArtifactCardShape.tsx`: a PURE editor-context-free `HowThisWasBuilt` export — a native `<button aria-expanded aria-controls>` disclosure (keyboard-operable, Focus Halo via `--Surface-Halo-Gap`) surfacing the trace fields inside a `<Surface mode="subtle">`, all `cardChrome`/token styling, all strings run through a `toPlainText` markup stripper (T-04.2-12), renders NOTHING when `agentTrace` is undefined. Added an `agentTrace: AgentTraceT | null` prop (`T.jsonValue` like `ir`, default `null`) and rendered `HowThisWasBuilt` inside the live `ArtifactCardBody`.
- `ArtifactCardShape.module.css` (new): token-only `:focus-visible` 2-layer Focus Halo for the disclosure toggle (inline CSSProperties can't express `:focus-visible`).
- `useExperienceLabRun.ts`: thread `result.agentTrace` from the result frame into the placed artifact card's props (completes the canvas link).
- `run/route.ts`: persist `run.agentTrace` on `persistArtifact` and carry it on the terminal result frame — the full workflow→route→Convex + workflow→route→canvas link.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] `useExperienceLabRun.ts` constructed `ArtifactCardShapeProps` without the new required `agentTrace` field**
- **Found during:** Task 3 (platform typecheck after adding the `agentTrace` prop to `ArtifactCardShapeProps`).
- **Issue:** Adding the required `agentTrace: AgentTraceT | null` prop to the shape made the existing `placeArtifactShape` props literal (useExperienceLabRun.ts:183) a type error (TS2741 missing property).
- **Fix:** Threaded `agentTrace` end-to-end: `placeArtifact(result)` reads `result.agentTrace ?? null`, `placeArtifactShape`'s `preview` param carries it, and the card props spread it. This also completes the canvas half of the AGENT-01 key-link.
- **Files modified:** `useExperienceLabRun.ts`.
- **Commit:** `bbced848`.

**2. [Rule 2 - Missing critical functionality] The route did not persist or carry the assembled trace**
- **Found during:** Task 3 (the plan's `key_links` require workflow finalizeRun → persistArtifact agentTrace AND runStream RunResultFrame → ArtifactCardShape agentTrace; the route is the seam that closes both, but the plan's file list only named the schema/mutation/workflow + the canvas UI).
- **Issue:** Without threading `run.agentTrace` through `run/route.ts`, the assembled trace would never reach Convex storage nor the canvas card — the persisted field + the card prop would always be empty in production.
- **Fix:** `run/route.ts` now passes `agentTrace` into the single-artifact `persistArtifact` call (additive, guarded) and onto the terminal `RunResultFrame`. The carousel-frame persistence path is intentionally NOT given a unified trace (per-frame artifacts have no single agent trace — out of scope, FND-03 honesty).
- **Files modified:** `apps/platform/src/app/api/experience-lab/run/route.ts`.
- **Commit:** `bbced848`.

**3. [Rule 1 - Bug] Fixture markup-safety + single-match contract failed against naïve React-escaped rendering**
- **Found during:** Task 3 (canvas.test.tsx AGENT-03).
- **Issue:** (a) React escaping leaves `onerror=` literally present in `innerHTML` for a smuggled `<img onerror=…>` headline — the fixture asserts `innerHTML` does NOT contain `onerror=`. (b) Listing the design rec's components inline produced TWO text nodes matching `/Surface/` (design row + registry-matches row), breaking the fixture's single-match `getByText(/Surface/)`.
- **Fix:** Added a `toPlainText` defence-in-depth stripper (removes angle-bracket constructs) applied to every displayed trace string (T-04.2-12); rendered the design-rec row as section+surfaceMode only so `Surface` appears in exactly one text node (the canonical Registry-matches row).
- **Files modified:** `ArtifactCardShape.tsx`.
- **Commit:** `bbced848`.

**Note (not a deviation):** `cardChrome.ts` was listed in the plan's `files_modified` but needed no change — its existing token fragments (`cardLabel`/`cardMeta`) were reused as-is by the trace card.
