# Phase 04.2 Plan 01: Wave 0 Failing-Test Scaffolds Summary

Established the executable RED acceptance contract for the entire 04.2 phase across six test files — layout primitives, the deterministic structural quality gate, advisor threading + backfill provenance, and the AGENT-02/AGENT-03 transparency UI — so every downstream Waves 2–5 task verifies against a test that already exists (Nyquist rule).

## What Was Built

Six `*.test.ts(x)` files extended with new `describe`/`it` blocks that fail now because the production code they assert does not exist yet. No production source was touched (verified by `git diff --name-only`).

### Task 1 — IR-core (`schema.test.ts`, `irToAst.test.ts`)
- `JioIRLayoutNode` block: (a) stack node with token gap + empty children parses; (b/b2) `tag`/`rawHtml` keys rejected by `.strict()`; (c) literal `gap:'16px'` rejected, (c2) `var(--Spacing-4)` accepted (LAYOUT-03); (d) responsive `columns:{S,L}` parses (LAYOUT-04); (e) nested layout-in-layout-in-instance to arbitrary depth (LAYOUT-02); (f/f2) old flat IR round-trips, new additive `layout` tree parses (Pitfall 1).
- `layout primitive compile` block: stack→`Container` (`direction:'column'`), grid→`Grid` (responsive `columns`), no `element`/`Stack`/`Row`/`Spacer` emitted (LAYOUT-05), old-flat-IR round-trips unchanged.

### Task 2 — validator + agents (`astValidator.test.ts`, `irGenerator.test.ts`)
- `validateIRStructure(ir, backfilled[])` IR-pre-check: `flat-layout` blocks a single flat Stack of leaves and clears once a depth≥2 layout exists; `empty-section-copy` blocks a section with no substantive copy.
- `placeholder-content` AST string-prop path: `placehold.co` URL, `'<Section> item'` humanized placeholder, and a threaded backfilled CONTENT prop all block; a backfilled STRUCTURAL prop only flags.
- `STRUCTURAL_LAYOUT_TYPES` allow: `Container gap:'4'` passes (skips empty-meta allowlist), `gap:'16px'` still blocks via `checkLiteralHook`, an unregistered layout-child still blocks (REG-03 not exempted).
- `irGenerator`: `GenerateIRInput` carries `designSpecs`/`copySpecs`/`messageHierarchy` consumed by `buildSectionPrompt`; `backfillRequiredProps` returns a `backfilled[]` `{instanceId, propName, isContent}` provenance list and never overwrites a model-supplied prop (QUAL-01/02/03).

### Task 3 — transparency UI (`runTurn.test.tsx`, `canvas.test.tsx`)
- `runTurn.test.tsx`: each pipeline step (`plan`/`design`/`copy`/`generate-ir`/`compile`/`validate`/`evaluate`) renders its human-readable `STEP_LABELS` activity line (not the raw token); an unknown step falls back to the raw token (AGENT-02).
- `canvas.test.tsx`: a `HowThisWasBuilt` helper renders an expandable "How this was built" `<button aria-expanded aria-controls>` disclosure surfacing the `agentTrace` fields, escapes markup (text-only, T-04.2-01b), and renders nothing when `agentTrace` is undefined (AGENT-03).
