  <options>
    <option id="postmessage-rects">
      <summary>data-ir-node-id attributes + postMessage rect report</summary>
      <pros>Reuses the existing iframe preview; node ids already exist in the IR; the iframe reports per-node bounding rects via postMessage to the canvas, which draws the overlay outside the iframe (no CSP relaxation).</pros>
      <cons>Requires a postMessage channel contract + a rect-reporting script inside the compiled bundle; rects must re-report on resize/scroll; adds a small per-node DOM attribute to compiled output.</cons>

      <cons>The canvas cursors (a user-explicit want, kept in-scope) ship later.</cons>
