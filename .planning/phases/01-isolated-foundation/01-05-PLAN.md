<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser canvas → run route (Node) | The browser posts a prompt-card config and consumes an event stream; orchestration is server-side. |
| Lab route ↔ existing Builder | The route group, layout, and canvas store are isolated; no shared layout, no `FoundationStyleProvider` touch, no Builder import. |
| Run route → Mastra | The single app-side Mastra invocation; Node runtime, never Edge. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-15 | Denial of Service (to existing app) | Isolation leakage from the Lab route | mitigate | Isolated layout (no shared `(builder)` layout / `FoundationStyleProvider`); scoped tldraw store; eslint boundary; grep gates assert no Builder import (LAB-03, Pitfall 3). |
| T-01-16 | Tampering / Elevation | Artifact card rendering raw markup | mitigate | Artifact card renders a structured IR summary from validated IR fields only — NOT a DOM preview / `dangerouslySetInnerHTML` (Claude's-discretion + IR-02). |
| T-01-17 | Tampering | Gap path silently producing an artifact | mitigate | Gap event short-circuits: card flips to typed gap state, NO artifact card created; jsdom test asserts zero artifacts on gap (FND-03/REG-03). |
| T-01-18 | Information Disclosure | Run route on Edge / leaking tokens | mitigate | Node runtime (Mastra requirement); the run route carries no auth/Convex tokens into any preview (preview is P3); `maxDuration` set. |
</threat_model>
