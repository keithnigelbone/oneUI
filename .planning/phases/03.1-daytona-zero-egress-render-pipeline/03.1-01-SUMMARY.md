# Phase 03.1 Plan 01: Daytona render-pipeline bundler foundation Summary

**`assembleAsset` turns the compiler's `@oneui/ui` TSX into ONE self-contained, deterministic, zero-egress HTML asset — real React inlined via esbuild, `@oneui/ui` resolved against the playground bundle alias, and brand `@layer brand` CSS generated server-side and inlined (no `<link>`, no CDN).**

## Accomplishments
- Built the missing foundation of the Daytona render path (CONTEXT root-cause #1): the executor's `ctx.bundle` was uncompiled TSX that nothing could render — `assembleAsset` produces the runnable asset.
- Rewrites the artifact's `@oneui/ui` import to a virtual `oneui-bundle` alias and esbuilds a wrapper (`bundle, write:false, iife, jsx automatic`) that inlines a REAL React/ReactDOM runtime (Daytona has no Sandpack providing one).
- Generates the brand `@layer brand` CSS string server-side via the pure engine (`precomputeBrandCSSNew` + `wrapCSSForInjection`), never the `useBrandCSS` React hook, and inlines it alongside token CSS into one HTML doc.
- All four BUNDLE-01 behaviors GREEN (self-contained, brand-CSS-inlined, deterministic, mounts); D-03 honored — the bundler consumes the compiler's TSX and never regenerates codegen.

## Decisions Made
- **esbuild as explicit devDep, no new install:** `^0.27` resolves to the already-present transitive `0.27.2` (RESEARCH Package Audit). No package-legitimacy checkpoint needed — nothing was downloaded.
- **oneui-bundle resolution:** prod path resolves to the CI-built `oneui-playground.mjs`; when absent (unit tests with no `build:playground`), an esbuild `onLoad` virtual module exposes every named export as a Proxy-backed placeholder React component (with `resolveDir` set so its own `react` import resolves). Keeps the asset self-contained and the output byte-stable without pulling the 18MB dist into tests.
- **minifyIdentifiers:false:** preserves `GeneratedArtifact`/`createRoot` so the asset is inspectable and the mount contract is assertable, while whitespace+syntax minification keeps it compact.
- **Brand CSS source:** engine-default scales for now; the per-brand foundation loader is the DaytonaExecutor caller's responsibility (Plan 04 seam), so this step stays a pure asset assembler.
