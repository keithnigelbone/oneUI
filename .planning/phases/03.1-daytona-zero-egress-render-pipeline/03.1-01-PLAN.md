<objective>
Build the missing foundation of the Daytona render path: a Lab-server bundler step
that turns a generated artifact (the compiler's TSX source, which imports `@oneui/ui`)
into ONE self-contained, no-external-imports HTML asset that mounts and renders with
zero network access. This is requirement BUNDLE-01.

The root cause of the deferred Daytona path (CONTEXT root-cause finding #1) is that
`ctx.bundle` is UNCOMPILED TSX source — `import { Button } from '@oneui/ui'; <Button/>`.
Nothing can render it. This plan produces the runnable asset the sandbox executes.

Per D-03 the bundler does NOT regenerate codegen: it consumes the compiler's TSX as
input, rewrites its `@oneui/ui` import to the pre-built playground bundle, supplies
real React/ReactDOM, inlines the brand CSS for `brandId`, and esbuilds the wrapper to
a single HTML string. RESEARCH Pattern 1 + Pattern 2.

Purpose: replace the source-string "bundle" with an executable asset so the sandbox
has something real to render and screenshot.
Output: `bundler/assembleAsset.ts` + tests; `esbuild` pinned as an explicit devDep of
the preview package (already present transitively at 0.27.2 — no new download).
</objective>
