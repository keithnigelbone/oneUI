<objective>
Fix the bug CONTEXT calls out ("Bug to fix alongside") and RESEARCH Pitfall 5: a
preview-executor failure currently surfaces on the canvas card as "gap — generation
refused", which is misleading because generation succeeded and only the
screenshot/preview (infra) step failed.

Root cause (verified in workflow.ts): when `previewStep` throws, the run-level `catch`
sets `outcome='error'`; then `finalizeRun` (line 758) demotes it to `'gap'` because
`ctx.ir` exists but `ctx.previewState?.url` is empty. The card then reads the gap copy.

Fix: catch the preview-executor failure AT the preview step, record a distinct
`previewError` signal on the context, and have `finalizeRun` surface it as a
preview-error (NOT 'gap') while leaving genuine DS gaps (uncovered foundation / failing
validation short-circuits) untouched. This must NOT touch the seam signature, the
compiler, the (builder) app, or the LabAstExecutor default (scope fence).

Purpose: correct failure attribution so a Daytona/preview infra failure is diagnosable.
Output: a `previewError` flag on the run context/result + finalizeRun polarity fix + a
regression test asserting the preview-error-vs-gap polarity.
</objective>
